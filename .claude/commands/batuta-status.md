Print a cross-project status snapshot derived from the vault and from each project's git state.

Argument syntax: `/batuta-status [--scope project|all|client:<slug>]`. Default `--scope all`.

Steps:

1. Resolve scope from `$ARGUMENTS`:
   - `--scope project`: only the current cwd's git repo (must have `.claude/kb-config.json`).
   - `--scope all`: every directory under `<vault_root>/clients/*/projects/*/` that has a corresponding repo on disk.
   - `--scope client:<slug>`: every project under that client.
2. Read `<vault_root>` from the current project's `.claude/kb-config.json`. If absent, ask the operator for the path or default to `~/batuta-kb`.
3. For each project in scope, gather:
   - **Branch** (current): `git -C <repo> rev-parse --abbrev-ref HEAD`
   - **Commits last 7 days**: `git -C <repo> log --since='7 days ago' --oneline | wc -l`
   - **Today's journal exists?**: `test -f docs/sessions/<today>-*.md`
   - **Drafts pending review**: `find <vault_root>/clients/<c>/projects/<p> -name '*.md.draft' | wc -l`
   - **Active plan**: first file in `docs/plans/active/`, if any
4. Cross-project aggregation:
   - **Total drafts pending**: across all clients
   - **Stale gotchas (>4 months)**: `find <vault_root>/gotchas <vault_root>/clients/*/projects/*/gotchas -name '*.md' -exec grep -l 'last_verified: 202[0-9]' {} \;` — filter by date arithmetic
   - **`_inbox/` size**: `find <vault_root>/_inbox -type f | wc -l`
5. Print a compact summary (≤30 lines):
   ```
   3 proyectos activos esta semana
   ─ bato-cajas (D:\bato-cajas) · feature/conciliacion-v2 · 5 commits 7d · journal hoy ✓ · 2 drafts pending
   ─ BATO2 (D:\BATO2) · main · 1 commit 7d · journal hoy ✗
   ─ batuta-portal · feature/v3 · 12 commits 7d · journal hoy ✓ · 1 draft pending

   Drafts pendientes review: 3
   Gotchas stale (>4 meses): 4
   _inbox sin clasificar: 12 archivos
   ```

Constraints:
- Read-only. Do NOT modify the vault or any project.
- Fail soft: if a project's `.claude/kb-config.json` is malformed, skip it with a warning. Do not abort the whole report.
- Output is for the chat, not for a file. For a deep weekly view, see `<vault_root>/STATUS.md` (regenerated by the cron routine).

See ADR-0011 D2 for the visibility surface design (3 surfaces: this slash, STATUS.md cron, _dashboard.md Dataview).
