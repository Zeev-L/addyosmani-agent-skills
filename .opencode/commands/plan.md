Load and follow the `planning-and-task-breakdown` skill using the native `skill` tool.

Read the existing spec (`SPEC.md` or equivalent) and relevant codebase sections. Then:

1. Stay in read-only planning mode
2. Identify dependencies between components
3. Slice work vertically instead of by horizontal layers
4. Write tasks with acceptance criteria and verification steps
5. Add checkpoints between phases
6. Present the plan for human review

Save the plan to `tasks/plan.md` and the task list to `tasks/todo.md`.
