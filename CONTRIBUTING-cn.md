# 为 Agent Skills 做贡献

感谢你对本项目的兴趣！这个项目是为 AI 编码代理提供的一套生产级工程技能集合。

## 添加新技能

1. 在 `skills/` 下创建一个使用 kebab-case 名称的目录
2. 按照 [docs/skill-anatomy.md](docs/skill-anatomy.md) 中的格式添加 `SKILL.md`
3. 包含带有 `name` 和 `description` 字段的 YAML 前言
4. 确保 `description` 简要说明技能的功能（第三人称），然后包含 `Use when` 触发条件

### 技能质量门槛

技能应该：

- **具体**——可操作的步骤，而不是模糊的建议
- **可验证**——带有证据要求的明确退出标准
- **经过实战测试**——基于真实的工程工作流，而不是理论理想
- **精简**——仅包含正确引导代理所需的内容

### 结构

每个新技能必须具有：

- 技能目录中的 `SKILL.md`
- 带有有效 `name` 和 `description` 的 YAML 前言

新技能通常应遵循标准结构：

- **概述**——此技能做什么以及为什么重要
- **何时使用**——触发条件
- **流程**——逐步工作流
- **常见合理化借口**——代理用来跳过步骤的借口，以及反驳
- **危险信号**——技能被错误应用的警告信号
- **验证**——如何确认技能被正确应用

### 不应该做什么

- 不要在技能之间复制内容——改为引用其他技能
- 不要添加模糊建议而不是可操作流程的技能
- 除非内容超过100行，否则不要创建支持文件
- 不要将参考材料放在技能目录中——改用 `references/`

## 修改现有技能

- 保持变更集中且精简
- 保留现有结构和语气
- 编辑后测试 YAML 前言是否保持有效

## 报告问题

如果你发现以下问题，请打开 issue：

- 给出不正确或过期指导的技能
- 常见工程工作流的缺失覆盖
- 技能之间的不一致
