## Decisions

- **Bundle PR #9 + PR #10 separately** despite both being incremental enforcement work, because v2.5 contracts (audit-chain Step 0, research-first Step 2 in implementers, batuta-agent-authoring rules 5–6) needed to land first so the v2.6 agent-architect baking and static validators could reference the canonical wording. Stacked-PR pattern: `fix/bundle-completo` was branched from `fix/audit-scope-and-research-first`, then rebased onto `origin/main` after PR #9 merged.
- **Static contract validators chosen over runtime E2E tests** for v2.6. Initial plan called for a `claude` CLI E2E harness; rejected as flaky (model latency, CLI version drift, stochastic regex matches). Static `grep`-based checks against agent prompt files are deterministic, fast, and CI-friendly. ADR for this lives implicitly in `tests/v2.5-validators/README.md`. If a runtime harness is added later, it complements rather than replaces the static suite.
- **`/save-plan` slash command chosen over `PreToolUse` hook on `ExitPlanMode`** for plan-mode persistence. Hook surface analysis showed `ExitPlanMode` tool input does not expose the plan file path, forcing a glob scan of `~/.claude/plans/*.md` for the most-recently-modified file — fragile under concurrent sessions and clock drift. ADR-0005 documents the analysis. v2.7 may revisit if Claude Code exposes the plan path in tool input.
- **`gh pr merge` blocking hook deferred to v2.7+**. Considered for v2.6 (would enforce "Claude never merges PRs" at runtime). Skipped because the rule is already documentary and the operator merges via web UI / terminal outside Claude — the hook would only fire on main's `gh` calls and would not block the actual violation surface. PRD entry records the rationale.
