# Session journal — 2026-04-29 — v3.8 Sprint 0 — authoring gates enforcement + KB pipeline plan

## Context

Strategic critical-thinking session that started as "evaluate Notion vs Obsidian, KB persistence reality, hook actually firing" and converged on a 4-sprint plan with Sprint 0 shipped in this session. Entry point: a green tree on `main` (last commit `2aeeb99` housekeeping for v3.7 journal).

The session went through 5 plan-mode iterations with operator feedback at each round before approval. The trigger for the final iteration was the operator's observation that the system permits a `MUST trigger X` declaration in CLAUDE.md to be violated by the very agent that authored the rules — proven empirically when the main agent itself proposed new skills/agents in the plan v2 without invoking `batuta-skill-authoring` or `batuta-agent-authoring`. The fix had to be runtime enforcement, not exhortation.

## Decisions

- **Vault primario migra a Obsidian** (`E:\Gdrive Batuta\My Drive\BATUTA AI\OBSIDIAN\BATUTA`, preexistente). Notion pasa a optional cliente-facing. Reasoning: pain point real es "conectar puntos cross-project" (Prophet/SAP vs Prophet/ICG); backlinks emergentes de Obsidian + grep local resuelven mejor que relations de Notion; operator's certification roadmap (probable ISO 27001 / SOC 2) favors local-first.
- **Persistencia automática vía git `post-commit` hook**, NOT Stop hook. Reuse of ADR-0005 reasoning for plan-mode persistence: Stop fires on every turn; commit is the intentional event. ADR-0005 is **superseded in part** (plan-mode `/save-plan` still applies); ADR-0011 covers session-journal persistence.
- **Vault structured in 4 levels (L0-L3) with explicit L1→L2 curation.** Captura sin curación es ruido — supersede chains coexisting at L2 produce ambiguous lookups. The `kb-curate` skill (Sprint 2) gates promotion with a 7-category classification matrix and hybrid control (decisions/playbooks/gotcha-update manual; gotcha-new/glossary auto). Marked-by-bullet frontmatter (`curated_into:`) for idempotency.
- **`research-first-dev` gains Step 1.5: Local KB lookup** prioritizing L2 > L3 > L1 with disclaimer for L1 hits. Staleness policy by `last_verified` age: <4mo cite local alone; 4-12mo dual cite; >12mo Context7 + refresh `last_verified`. Sprint 2.
- **Two MUSTs converted from text to runtime enforcement** (D3 of ADR-0011 — this Sprint 0 PR): `skill-authoring-required` + `agent-authoring-required` rules + `pre-write-{skill,agent}-gate.sh` hooks + 11 test cases. Marker workflow: skill leaves `.claude/.authoring-marker-{skill,agent}-<ISO>` (60-min TTL) as final step; hook checks marker freshness, walks up to find `.claude-plugin/`, verifies origin regex, blocks creation otherwise. Bypass envs scoped per gate (`BATUTA_SKILL_AUTHORING_BYPASS=1` / `BATUTA_AGENT_AUTHORING_BYPASS=1`). Project-local `.claude/agents/` paths skip origin check (rule applies to any repo that imports it).
- **`agent-architect` Phase 5.0 added** to write the agent marker before specialist creation. The runtime exception in the rule keeps both surfaces (plugin-shipped agents under `agents/` and project-local specialists under `<project>/.claude/agents/`) symmetric — same marker file, same hook.
- **Six discovery decisions** for the KB candidates from the v3.8 plan: all 6 produced **CREATE** verdicts. No reuse, no fork — distinctness justified per candidate in ADR-0011.
- **User-global hook registration deferred to post-merge.** Registering the new hooks in `~/.claude/settings.json` AS the same session that wrote them would self-block the remaining file creations. The post-merge step is a manual operator action (run `/plugin update batuta-agent-skills` to load the new hooks via the marketplace install path).
- **PR was misrouted to upstream** (`addyosmani/agent-skills#116`) on first attempt — `gh pr create` without `--repo` defaults to the parent repo of the fork. Closed with comment, reopened correctly on `jota-batuta/batuta-agent-skills#31`.
