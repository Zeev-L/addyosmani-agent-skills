# Session journal — 2026-05-04 — v4.0 distillation

Entry point: `docs/plans/active/2026-05-02-kb-obsidian-graph-fix.md` and the operator-pasted Ultraplan refinement at `~/.claude/plans/here-is-a-draft-wondrous-neumann.md` (saved manually as `bueno-graphify-ten-a-dos-eventual-treasure.md` was the original Plan-mode draft; the Ultraplan refinement added Slice 0 and exhaustive file enumeration for Slice 1).

## Context

The plugin reached v3.8 with 36 skills + 9 agents + 8 rules + 7 hooks + 15 validators. A diagnostic identified four classes of debt:

- graphify dual-engine narrative was fiction on Windows (always BROKEN per upstream issue #378).
- `/code-simplify` produced bad results because the command entry point lacked scope and readiness checks.
- `simplify-ignore` hook (302 lines) was authored but never registered or used.
- `notion-kb-workflow/` directory remained on disk despite ADR-0012 deprecation.
- 6 of 8 rules titled "Inviolable" had no runtime hook backing them — overpromise.
- `setup-rules.sh --all` chained into `setup-code-graph.sh`, coupling rule import to engine bootstrap.
- PRD `Last reviewed` 11 minor versions stale; SPEC validator count 6 cases stale; DELEGATION-RULE missing 3 KB agents.

## Decisions

- **graphify deprecated, not removed transitively** ([[ADR-0013]]). Single-engine code-graph from v4.0 onward (codebase-memory-mcp). The `graphify claude install` prohibition stays because the upstream command still exists as a footgun. ADR-0007 preserved unchanged as historical record.
- **State file backward-compat preserved.** The `graphify` key in `~/.claude/code-graph-engines.json` is now `"status": "DEPRECATED"`; older cached `check-code-graph-engines.sh` versions continue to parse the file without error.
- **simplify-ignore deleted, not moved to `_deprecated/`**. Git history is the archival surface. The plugin is small enough that an "attic" directory is more confusing than helpful.
- **research-first-dev + source-driven-development stay separate.** Distillation analysis showed orthogonal triggers (per-import vs per-pattern). Merging would couple unrelated concerns.
- **KB pipeline (3 agents) stays as-is.** The architecture is sound; what it needs is vault content over time, not redesign. The kb-curator / kb-pipeline / kb-backfiller distinction is intentional and documented.
- **Authoring gates stay.** Today's session validated their value: when the operator pasted the v3.8+ candidates list, the Ultraplan refinement caught and corrected the v3.8 → v4.0 versioning split. The same pattern of "vague directives execute blind" is what the gates prevent.
- **Commit one slice at a time.** 8 commits on `chore/v4.0-distillation`, each verifying the validator suite stays green.
