# Batuta status

## Overview

One vault, multiple projects, multiple clients — cross-project visibility degrades fast without a dedicated surface. This skill aggregates git state and vault metadata across all in-scope projects and prints a compact summary (≤ 30 lines) to the chat. It is read-only and fail-soft: a malformed config in one project never aborts the whole report.

Three visibility surfaces exist (ADR-0011 D2): this slash command (ad-hoc), `<vault_root>/STATUS.md` (regenerated by the weekly cron), and `_dashboard.md` Dataview (Obsidian-native). This skill powers the first surface.

## When to Use

- At session start, to orient before picking up work.
- Any time the operator wants a bird's-eye view of all active Batuta projects.
- As a quick triage tool to spot stale gotchas, inbox backlog, or projects that missed their daily journal.

Argument syntax: `/batuta-status [--scope project|all|client:<slug>]`. Default: `--scope all`.

Do NOT use for deep dives into a single project — read that project's active plan and last session journal directly.

## Process

### Step 1: Resolve scope

Parse `$ARGUMENTS`:
- `--scope project`: only the current cwd's git repo (must have `.claude/kb-config.json`).
- `--scope all`: every directory under `<vault_root>/clients/*/projects/*/` that has a corresponding repo on disk.
- `--scope client:<slug>`: every project under that client slug.

### Step 2: Resolve vault root

Read `<vault_root>` from the current project's `.claude/kb-config.json`. If absent, ask the operator for the path or default to `~/batuta-kb`.

### Step 3: Gather per-project metrics

For each project in scope, collect:

| Metric | Command |
|---|---|
| Current branch | `git -C <repo> rev-parse --abbrev-ref HEAD` |
| Commits last 7 days | `git -C <repo> log --since='7 days ago' --oneline \| wc -l` |
| Today's journal exists? | `test -f docs/sessions/<today>-*.md` |
| Drafts pending review | `find <vault_root>/clients/<c>/projects/<p> -name '*.md.draft' \| wc -l` |
| Active plan | First file in `docs/plans/active/`, if any |

Fail soft: if a project's `.claude/kb-config.json` is malformed or the repo path does not exist, skip it with a one-line warning in the output. Do not abort.

### Step 4: Cross-project aggregation

Compute once across all in-scope projects:

- **Total drafts pending**: sum of `.md.draft` files across all vault locations.
- **Stale gotchas (> 4 months)**: scan `<vault_root>/gotchas` and `<vault_root>/clients/*/projects/*/gotchas` for files whose `last_verified:` frontmatter date is more than 4 months before today.
- **`_inbox/` size**: `find <vault_root>/_inbox -type f | wc -l`.

### Step 5: Print summary

Output is for chat only — do NOT write to a file.

```
3 proyectos activos esta semana
─ bato-cajas (D:\bato-cajas) · feature/conciliacion-v2 · 5 commits 7d · journal hoy ✓ · 2 drafts pending
─ BATO2 (D:\BATO2) · main · 1 commit 7d · journal hoy ✗
─ batuta-portal · feature/v3 · 12 commits 7d · journal hoy ✓ · 1 draft pending

Drafts pendientes review: 3
Gotchas stale (>4 meses): 4
_inbox sin clasificar: 12 archivos
```

Keep output ≤ 30 lines. For the weekly regeneration of `STATUS.md` in the vault, the weekly cron routine handles it.

## Red Flags

- A project shows 0 commits in the last 7 days but has an open plan in `docs/plans/active/` — slice may be stalled.
- `_inbox/` size > 20 — inbox drain is behind; trigger `/kb-curate --scope all-pending`.
- More than 50 drafts pending across all projects — operator is not reviewing drafts; surface the specific files.
- A project has no `.claude/kb-config.json` — it was not initialized with `batuta-project-hygiene`; show a warning per project.

## Verification

- Output contains one line per project in scope.
- Running `/batuta-status --scope project` from a repo with a valid `.claude/kb-config.json` returns exactly one project line.
- Running `/batuta-status --scope client:nonexistent-slug` returns an empty project list with a clear message, not an error.
- Re-running produces the same output for the same git and vault state (deterministic, read-only).
